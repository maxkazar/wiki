Value Object — шпаргалка

Компрессия Лессона 01. Держи под рукой на ревью.

Определение за одну строку

Доменное понятие без идентичности: важно только значение, оно иммутабельно и валидно по построению, а поведение едет вместе с данными.

Три обязательных свойства

1. Иммутабельность. freeze в конструкторе. Нужно другое значение → новый объект.
2. Валидность по построению. Конструктор поднимает ArgumentError на невалидном входе. Невалидный объект не существует.
3. Поведение с данными. Вопросы к значению (severity, critical?, contains?) — методы объекта, не внешние хелперы.

Скелет на Ruby (PORO поверх AR)

class CvssScore
  include Comparable
  attr_reader :value

  def initialize(value)
    @value = Float(value)
    raise ArgumentError unless (0.0..10.0).cover?(@value)
    freeze
  end

  def severity = ...          # поведение
  def <=>(o) = value <=> o.value   # порядок
  def ==(o) = o.is_a?(self.class) && value == o.value
end

Триггеры на ревью

  • Две колонки, которые могут противоречить (score+level) → объединить в Value Object.
  • Один и тот же case/валидация повторяется в нескольких местах → поведение «хочет» в объект.
  • Строка с ручной regex-валидацией в разных контроллерах → Value Object с валидацией в конструкторе.
Калибровка — не оверинжинирь Если «значение» используется в одном месте и не имеет поведения — может, хватит и колонки. Value Object окупается, когда есть поведение или риск невалидного/противоречивого состояния. См. контр-перспективу 37signals.

# Глоссарий: DDD в Rails (платформа ИБ)

Единый язык курса и команды (**ubiquitous language**). Одни и те же слова — в коде, на ревью, в разговоре с продактом. Если термин здесь — используем именно его, без синонимов.

> Соглашение: рядом с русским термином — английский канон DDD в скобках, потому что в коде имена классов/модулей пишем по-английски.

## Тактические паттерны DDD

| Термин | English | Определение | Лессон |
|---|---|---|---|
| Объект-значение | **Value Object** | Доменное понятие без идентичности: важно значение. Иммутабелен, валиден по построению, поведение едет с данными. Сравнение по содержимому. | [01](lessons/0001-value-object.html) |
| Сущность | **Entity** | Доменный объект с идентичностью (`id`): «тот же самый», даже если поля изменились. Сравнение по `id`. | [01](lessons/0001-value-object.html) |
| Агрегат | **Aggregate** | Кластер Entity и Value Objects, меняющихся вместе в одной транзакции. Граница консистентности. | [02](lessons/0002-aggregate.html) |
| Корень агрегата | **Aggregate Root** | Главная Entity агрегата и единственная точка входа. Снаружи держим ссылку только на корень (по `id`). | [02](lessons/0002-aggregate.html) |
| Инвариант | **Invariant** | Правило домена, истинное всегда. Корень проверяет его при каждой команде и отклоняет невалидное изменение целиком. | [02](lessons/0002-aggregate.html) |
| Команда | **Command** | Публичный метод корня, выражающий намерение домена (`topology.connect(a, b)`), а не сеттер. Внутри — проверка инвариантов. | [02](lessons/0002-aggregate.html) |
| Доменный сервис | **Domain Service** | PORO без состояния и без I/O для «ничьей» логики — деятельности домена, тянущей несколько агрегатов сразу (`AttackSurfaceCalculator`). | [03](lessons/0003-domain-service.html) |
| Прикладной сервис-объект | **Application Service** | Оркестратор: транзакция + вызов доменного кода + сайд-эффекты (письма, джобы). Доменных правил не содержит (`RunScan`). | [03](lessons/0003-domain-service.html) |
| Анемичная модель | **Anemic Domain Model** | Анти-паттерн: модель — голый держатель данных без поведения, вся логика сбежала в сервисы. Запах оверинжиниринга. | [03](lessons/0003-domain-service.html) |
| Ограниченный контекст | **Bounded Context** | Граница (= модуль), внутри которой термин имеет ровно одно значение. Проводится там, где смысл слова меняется. Через границу — id + публичный интерфейс/события. | [05](lessons/0005-bounded-context.html) |
| Карта контекстов | **Context Map** | Схема отношений между контекстами: кто кому upstream/downstream, как общаются. | [05](lessons/0005-bounded-context.html) |
| Анти-коррупционный слой | **Anti-Corruption Layer (ACL)** | Фасад/адаптер на границе с внешней системой: переводит чужую модель в твою, чтобы её словарь не протёк в домен. Только перевод, без бизнес-правил. | [07](lessons/0007-anti-corruption-layer.html) |
| Adapter / Translator / Facade | — | Три слоя ACL: протокол (HTTP/auth/ретраи) / маппинг чужого формата в домен / публичный порт в языке домена. | [07](lessons/0007-anti-corruption-layer.html) |

## Домен: платформа ИБ

| Термин | English / код | Определение |
|---|---|---|
| Актив | **Asset** | ⚠️ Контекст-зависим: в Топологии — `Node` (узел графа), в Уязвимостях — `ScanTarget` (цель скана), в Документации — `AssetEntry` (запись отчёта). Связаны общим `asset_id`, а не одной моделью. См. [Лессон 05](lessons/0005-bounded-context.html). |
| Топология | **Topology** | Граф ИС: узлы и связи между ними. Агрегат; корень — граф. |
| Узел | **Node** | Вершина графа топологии. Entity внутри агрегата Topology. |
| Связь | **Link** | Ребро между двумя узлами. Внутренний объект агрегата Topology. |
| Скан | **Scan** | Прогон анализа уязвимостей. Кандидат в корень агрегата. |
| Находка | **Finding** | Конкретная уязвимость, обнаруженная на активе/узле. Ссылается на узел по `node_id` (через границу агрегата). |
| Уязвимость | **Vulnerability / CVE** | Известная уязвимость из каталога (CVE-ID). |
| CVSS-оценка | **CvssScore** | Value Object: значение 0.0–10.0 + severity-поведение. |
| Критичность актива | **AssetCriticality** | Value Object: уровень важности актива (low…critical). |
| Документ | **Document** | Генерируемая документация по ИС. Кандидат в корень (Document → Section). |

## Правила языка

- **Не плодим синонимы.** «Находка» = Finding всегда; не «алерт», не «issue», не «проблема».
- **Сначала глоссарий, потом код.** Новое понятие домена → сперва строка здесь, потом класс.
- **Имена классов = термины.** `Topology::Graph`, `Security::CvssScore`, `Finding` — читаются как глоссарий.

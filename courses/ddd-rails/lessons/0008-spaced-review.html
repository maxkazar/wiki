Лессон 08 · DDD в Rails · Повторение (storage strength)

Повторение с интервалом

Смешанный прогон по всем семи лессонам. Цель — не «вспомнить сейчас», а чтобы знание осело надолго. Усилие припоминания здесь — это фича, не баг.

Как извлечь максимум Этот лессон работает лучше всего через паузу после остального курса — день, а лучше два-три. Тогда припоминание даётся с усилием, и именно это усилие переводит знание в долгую память (spaced retrieval). Он живёт в воркспейсе — возвращайся и проходи его повторно через неделю, через месяц. Каждый повтор укрепляет след.

Часть 1 · Свободное припоминание

Не подглядывая и без вариантов — проговори ответ вслух или запиши, и только потом разверни сверку. Свободное припоминание сильнее узнавания.

Вспомни 4 уровня лестницы «куда положить логику» (Л03–04)
  • 1. Про одно значение → метод объекта-значения.
  • 2. Про состояние одного агрегата + инвариант → команда корня.
  • 3. «Ничья», несколько агрегатов, без I/O → доменный сервис.
  • 4. Оркестрация + сайд-эффекты → прикладной сервис-объект.
  • 0. Запах: логика читает поля одной модели и кладёт назад → вернуть на модель.
Вспомни 3 обязательных свойства Value Object (Л01)
  • Иммутабельность — создан и не меняется (freeze).
  • Валидность по построению — невалидный объект не существует (конструктор падает).
  • Поведение едет с данными — вопросы к значению это его методы.
Вспомни 3 слоя анти-коррупционного слоя (Л07)
  • Adapter — протокол: HTTP, авторизация, ретраи.
  • Translator — чужой формат → доменные объекты (тут умирает чужой словарь).
  • Facade / порт — публичный вход в языке твоего домена.
Вспомни, где проходит граница ограниченного контекста (Л05)
  • Там, где смысл термина меняется («нужна другая модель, когда меняется язык»).
  • Через границу — id + публичный интерфейс/события, не лазая в чужие внутренности.
  • Не дроби, где слово значит одно и то же; не объединяй, где значит разное.

Часть 2 · Смешанные кейсы (все 7 лессонов)

Перемешано намеренно. Реши сам — потом жми и читай фидбэк. Счёт сверху.

Что из перечисленного — объект-значение?

CvssScore. Нет идентичности, важно значение, поведение (severity) едет с данными. Scan и Document — Entity: у них есть «тот же самый» по id.

Что сигналит, что между двумя модулями проходит граница контекста?

Слово меняет смысл. Граница контекста — там, где сдвигается язык. «Актив» как узел графа vs как запись отчёта — два значения, два контекста.

Зачем анти-коррупционный слой на интеграции со сканером?

Чтобы чужая модель не протекла. Без ACL словарь сканера (plugin_id, risk_factor) «разлагает» чистый домен. Перевод происходит один раз, на границе.

Кто гарантирует инвариант «нет висячих связей» в графе топологии?

Корень агрегата. Инвариант охраняет единственная дверь — topology.connect. Размазать проверки по контроллерам = гарантий нет.

RunScan: дёргает внешний API, сохраняет находки, шлёт уведомление. Это…

Прикладной сервис-объект. Внешний I/O + запись + уведомление — оркестрация, доменных правил нет. Не путать с доменным сервисом (чистый расчёт).

«Актив» нужен и Топологии, и Документации. Как моделировать?

Две модели, общий id. Каждый контекст держит свою модель «Актива» под свой язык. Единая бог-модель — унификация, которая «не окупается».

Перевод формата Nessus в доменный Finding живёт в…

В транслейторе. Adapter — только протокол; Translator — маппинг чужого JSON в домен. Finding о чужом формате не знает.

Сервис читает поля одной Finding, считает число и кладёт назад. Реакция?

Вернуть на модель. Запах анемии: поведение сбежало от своих данных. Один агрегат + чтение себя = метод модели, не сервис.

Документации нужны результаты скана из контекста «Уязвимости». Как?

Фасад или событие. Контекст не лезет в чужие внутренности — «модуль не зовёт код другого модуля». Просим через одну дверь или подпиской на ScanCompleted.

Что НЕ должно жить внутри анти-коррупционного слоя?

Бизнес-правила. ACL — только перевод (+ валидация на границе доверия). Логика домена остаётся на агрегатах и сервисах.

Finding ссылается на узел из агрегата топологии через…

asset_id. Через границу агрегата/контекста — только ссылка по id, не AR-ассоциация, которая тянет и связывает чужой агрегат.

Сменили сканер Nessus → OpenVAS. Что приходится переписать?

Адаптер и транслейтор. Домен зависит от порта VulnerabilityScanner в своём языке, а не от вендора. В этом весь выигрыш ACL.

Куда дальше расти Знаниевая база командного стандарта собрана. Самый сильный следующий шаг — навык на живом коде: принеси на разбор реальный модуль или PR вашей системы, и прогоним его по чек-листам из лессонов. И поделись стандартом с командой — глоссарий и шпаргалки готовы к расшариванию.
